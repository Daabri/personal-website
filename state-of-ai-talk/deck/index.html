The State
of AI

What's been happening, in six steps. And where we take it next.

Daan  /  Product Design  ·  press → to begin
00:00 // The map

Six steps.
One direction.

AI didn't arrive in a thunderclap. It climbed a ladder, one rung at a time. Up top are all six. Each one changed the outcome, not just the tooling. The last one is what we're building now.

01
01 // Autocomplete

Everyone just
got faster.

It started quietly. Suggestions while you type. Emails, docs, code, spreadsheets, all a little quicker. Same job, less friction. Nobody's role changed. The warm-up.

02
02 // Agents that act

It stops suggesting.
It does.

Hand it a task, it does the task. For me that meant finally shipping my own PRs, the design engineer I always wanted to be. I built the shaders on the POS lock screen with Claude Code. Design that ships itself.

“Code must not be written by humans.”

StrongDM, an engineering principle

03
03 // Connecting tools

It works from our
world, not the web's.

Plug it into our actual tools and it stops guessing. The POS landing page came together from our own context: notes in Granola, specs in Jira, shaping in Confluence, code in GitHub. Wired up, it builds things that fit us.

04
04 // Skills

Do it once. Never
explain it again.

Anything you repeat becomes a skill the tools reuse. My SEO review runs as a skill now. The checklist I used to run by hand, captured once, applied every time. Marketing gets the same review on demand, no me required.

05
05 // Looping agents

You stop prompting.
You write the loop.

The newest rung. Instead of asking each time, you set up a loop that runs, checks its own work, and keeps going. Triage, build, review, on repeat, while you do something else.

“My job is to write loops.”

Boris Cherny, creator of Claude Code

// Reality check

Not all of it
is magic.

Some of it is genuinely new. Some of it is a for-loop with a confident voice and a large monthly bill. The skill is telling them apart. And most of the real work now is the unglamorous part: the guardrails that make it stop and check itself.

// Where it goes

“AI automates what you can verify.”

Andrej Karpathy

If you can check it, a machine can do it. If you can't, it stays human. Judgment, taste, direction. Still ours.

"Without human direction, you have compute running in circles."  — Satya Nadella

06
06 // MCP

One port.
Many cables.

The next rung, and the one we're building. Today every tool is wired up by hand, per project. MCP is one shared doorway any AI plugs into, for all our data, docs and tools. Build it once, everyone uses it.

Anthropic's USB-C for AI. An open standard. OpenAI, Google and Microsoft all use it.

06
06 // MCP, for us

Ask our data
a question.

It's already starting. We can reach Snowflake from inside Claude, Copilot and ChatGPT now. No SQL, no waiting on a dashboard. Anyone, any department, asks in plain language and gets an answer from our own numbers.

06
06 // MCP, outward

The same door
opens to partners.

A zoo's assistant, a park's tools, an operator's own systems, all able to plug into Convious. Availability, pricing, bookings, in real time, through one open connection. We stop being an app people log into and become the layer they build on.

Whoever owns the connective layer in a fragmented market owns the market.

// The takeaway
The tools keep changing. What compounds is our connected context, and the judgment behind it.

Every tool we connect and every skill we write down makes the next step easier. We're not chasing models. We're building the doorway.
